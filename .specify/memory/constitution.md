# Project Constitution: Hackathon II – Phase II | Production-Grade Full-Stack Todo Web Application

## 1. Core Philosophy

### 1.1 100% AI-Native & Spec-Driven Development
Zero manual coding — everything generated from detailed, versioned Markdown specifications

### 1.2 Modern Full-Stack Stack
Next.js 14+ (App Router + Server Components) • FastAPI (async + Pydantic v2) • PostgreSQL (Neon serverless)

### 1.3 Extreme Modularity
Each layer (UI, API, Auth, DB, Tests) independently spec-able, mockable & testable

### 1.4 Enterprise-Grade Security
JWT + Refresh Tokens • Argon2id hashing • OWASP Top 10 mitigation • HttpOnly + Secure cookies

### 1.5 Future-Proof Architecture
Stateless, horizontally scalable, prepared for Phase III AI agents & Phase IV Kubernetes + observability

## 2. Non-Negotiable Standards

### 2.1 Every feature must have
Constitution section + Detailed spec + Acceptance criteria + Edge cases + Wireframe (text/Mermaid) + OpenAPI schema

### 2.2 UI/UX Quality Bar
Mobile-first • Dark/Light mode • Tailwind + shadcn/ui • Framer Motion micro-animations • Accessible (WCAG 2.2 AA) • Keyboard-first navigation

### 2.3 Intermediate+ Features Mandatory
Priority (color-coded 4 levels), Multi-tags (chips + autocomplete), Full-text + fuzzy search, Multi-filter (status/priority/tags/date range), Multi-sort + drag-to-reorder

### 2.4 Advanced Touches
Optimistic updates • Undo toast • Infinite scroll/pagination • Command palette (⌘K) • Real-time validation

## 3. Strict Constraints

### 3.1 Tools only
Claude Code / Gemini CLI / Spec Kit Plus / Cursor / Aider

### 3.2 No hand-written code at any point
No hand-written code at any point

### 3.3 Timeline
Phase II MVP complete within hackathon window

### 3.4 Phase II excludes
AI task generation, real-time collab, notifications, recurring tasks

## 5. Governance

### 5.1 Versioning
The project constitution follows semantic versioning (MAJOR.MINOR.PATCH).
- **MAJOR**: Backward incompatible governance/principle removals or redefinitions.
- **MINOR**: New principle/section added or materially expanded guidance.
- **PATCH**: Clarifications, wording, typo fixes, non-semantic refinements.
Current Version: 0.1.1

### 5.2 Amendment Procedure
Any proposed amendments to this constitution MUST be reviewed and approved by the core architectural team. Changes will be discussed, documented, and ratified before being merged.

### 5.3 Compliance Review
Adherence to these principles and standards will be periodically reviewed as part of project milestones and code audits. Deviations MUST be justified and documented, with a clear plan for remediation.
